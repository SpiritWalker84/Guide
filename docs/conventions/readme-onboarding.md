# README генерируемого проекта

## Язык

- **README.md генерируется на русском языке.**

## Обязательные разделы (`readme_onboarding.required_sections_ru`)

В README следующего разработчика должны быть раскрыты темы (формулировки из `defaults.yaml`):

1. **«О проекте»** — зачем репозиторий, кому полезен, основные возможности и **явные ограничения** (что вне scope).
2. **«Стек технологий»** — язык и версия Python (или иной runtime), фреймворки, ключевые библиотеки, БД/кэш/очереди/бот API — всё, что **реально используется** в коде.
3. **«Структура репозитория»** — дерево или таблица основных каталогов и модулей, кратко что где лежит (точка входа, конфиг, доменная логика, интеграции).
4. **«Запуск»** — `docker compose` и при необходимости локальный запуск; порты и типовые команды.
5. **«Конфигурация»** — переменные из `.env.example`, что обязательно задать.
6. **«Разработка и тесты»** (если есть тесты или скрипты) — как гонять тесты/линтеры, полезные команды.

## Дополнительные требования из `prompt_rules`

- README должен **явно упоминать**: по умолчанию используется **HTTP**, если TLS не настроен отдельно.
- Текст README должен позволять **онбординг без чтения всего кода** — опираться на перечисленные разделы.

## Поведение генератора (оригинальный проект)

Если README получается «тонким», генератор может **дополнить** блоком «Структура репозитория» на основе сгенерированных путей (см. корневой `README.md` репозитория ProxyAI Code Generator).
