# Guide — конвенции для генерации и разработки кода

Публичный набор правил и чеклистов (OOP, модульность, env, русские docstrings, Docker/README, профили и т.д.). Репозиторий содержит **только документацию** в каталоге [`docs/conventions/`](docs/conventions/README.md).

**Канонический URL:** https://github.com/SpiritWalker84/Guide

---

## Как ссылаться из чата (Cursor, другой ассистент)

Скопируйте и при необходимости дополните задачей:

```text
Напиши код по указаниям из репозитория https://github.com/SpiritWalker84/Guide
Сначала прочитай оглавление и все релевантные файлы в docs/conventions/ (начни с docs/conventions/README.md).
Соблюдай эти правила полностью.

Перед реализацией: сделай детальный анализ ТЗ (цель, scope, неясности, риски). Если возможны разные способы реализации — задай вопросы пользователю в формате вариантов A) B) (и при необходимости C)…), у каждого варианта распиши плюсы и минусы; при необходимости укажи рекомендацию по умолчанию. Не переходи к массовому написанию кода, пока по спорным местам нет выбора или явного «сделай сам с обоснованием». Подробнее: docs/conventions/tz-analysis-workflow.md.

После существенной реализации: проведи код-ревью по docs/conventions/code-review.md (чеклист как у шага _review_chain в генераторе: зависимости, заглушки, запуск, секреты, README, стиль документации); критичные замечания устрани или явно оставь на согласование.

Инфраструктура запуска: для сервисов (веб, API, боты и т.п.) предусмотри docker-compose.yml и Dockerfile(ы); основной запуск в README — docker compose up -d (при первом разе с --build). Простой CLI или мелкий скрипт без compose — допустимо, если это следует из ТЗ или логики задачи (см. docker-and-compose.md).

Задача: <кратко опишите задачу>.
```

Короткий вариант:

```text
Код и структура — по гайду https://github.com/SpiritWalker84/Guide (docs/conventions/). Сервисы — docker compose up -d; простой CLI может без compose (docker-and-compose.md). Сперва анализ ТЗ; спорные места — A)/B) (tz-analysis-workflow.md); после кода — code-review.md. Задача: …
```

Для моделей с доступом к URL укажите ветку `main` и путь к файлам; при отсутствии доступа к GitHub приложите содержимое нужных `.md` или используйте «Add to Chat» / вложение файлов из локального клона.
